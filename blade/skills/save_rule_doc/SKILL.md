# save_rule_doc — 保存规则文件到规则库

## 何时触发
用户在聊天中**已附带上传一个文件**（文件已在当前 session 的 sandbox 中），并表达把它存为规则文件的意图，例如：
- "把这个存为规则文件"
- "加入规则库 / 保存到规则库"
- "这是一份规则文档，存一下"

## 前置判断
1. 确认 sandbox 中确有用户刚上传的文件，拿到它的绝对路径。
2. 判断它是否为**规则类文档**（政策、指南、规范、办法等）。
   - 若明显不是规则文档（随手的图片、无关表格等），**先说明并询问用户是否仍要保存**，不要自作主张入库。
   - 若用户意图不明确，先澄清再动作。

## 执行
对确认要保存的文件，运行（每个文件路径作为参数，可多个）：

```bash
smart-doc-add "<sandbox 中的文件绝对路径>"
```

`smart-doc-add` 会把文件上传到规则库后端并入库。

## 解读结果（按退出码）
| 退出码 | 含义 | 你应如何回报 |
|---|---|---|
| 0 | 全部成功 | 报告每个文件的标题与编号，例："已存入规则库：政策A（SD-abc123）" |
| 1 | 用法错误（没给文件） | 说明没有可保存的文件 |
| 2 | 文件不存在/不可读 | 说明没找到文件，请用户确认是否已上传 |
| 3 | 连不上规则库后端 | 说明后端暂时不可达，建议稍后重试 |
| 4 | 后端返回错误 | 说明入库失败（附 stderr 原因），不要声称成功 |
| 5 | 部分/全部文件失败 | 逐个报告：成功的报标题+编号，失败的报 stderr 里的原因（如超过大小上限） |
| 6 | 后端地址未配置 | 这是部署问题，提示需要配置 SMART_DOC_API |

## 铁律
- **只有退出码 0 且 stdout 出现 `doc_code=...` 才能说"已保存成功"。** 其他情况一律如实说明失败原因，绝不谎报。
- 不要凭空编造 doc_code 或标题——只转述 `smart-doc-add` 的实际输出。
