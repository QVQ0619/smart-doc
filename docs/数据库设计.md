# 数据库设计

> 项目：项目申报智能审查与仿真验证平台
> 来源文件：`docs/file/审查系统-schema_mysql.sql`（权威建表脚本）
> 数据库：`smart`（MySQL 8.0.17+，InnoDB / utf8mb4 / 时间统一存 UTC）
> 规模：共 **57 张表**，划分为 **11 个业务域**（5A–5K）

## 全局约定

| 项目 | 说明 |
|---|---|
| 引擎 / 字符集 | InnoDB，`utf8mb4`，`ROW_FORMAT=DYNAMIC` |
| 主键 | 统一 `id BIGINT AUTO_INCREMENT`（日志分区表主键含时间列） |
| 时间 | `DATETIME(3)`（毫秒精度），统一存 UTC，应用层转换 |
| 布尔 | `TINYINT(1)`（0/1） |
| 枚举 | `VARCHAR` + `CHECK` 约束 |
| 金额 | 定精度 `DECIMAL`（如 `DECIMAL(16,2)`） |
| JSON | 原生 `JSON` 类型，部分配 `JSON_SCHEMA_VALID` 校验 |
| 嵌入向量 | 关键检索表带 `embedding_id / embedding_model / embedding_version / embedded_at` 一组字段 |
| 软删除 | 关键表带 `deleted_at` |
| 日志表 | 按月 `RANGE` 分区、放弃外键（append-only） |

## 核心数据流（实体关系脉络）

```
机构/用户/角色权限(5A)
        │
政策文件 standard_doc → 条款 regulation_clause
        │                      │
审查维度/规则 review_rule → review_rule_version ──┐
        │                                          │
配置包 config_package ── config_rule_version ──────┘（锁定规则版本快照）
        │
审查批次 review_batch（绑定配置包）
        │
申报项目 declared_project → 申报包 application_package
                                   │
              ┌────────────────────┼─────────────────────┐
        材料文件 material_file   抽取字段/成员/预算/附件   复审轮次 review_round
              │                                              │
        解析片段 parse_segment ◄──(溯源)── 审查项 round_check
              │                              │      │
        证据 finding_evidence ◄─────────────┘      ├─ 复核动作 check_review_action
                                                    └─ 审查报告 review_report
                                   │
                          审查案例 review_case → 问题模式库 case_pattern
                                   │
              仿真：sim_experiment → sim_run（平台 sim_platform）
                                   │
        全程：file_object（文件桥）+ agent_run_log / model_call_log / audit_log（日志审计）
```

## 设计亮点

1. **规则版本化**：`review_rule`（身份）+ `review_rule_version`（不可变版本）+ `config_rule_version`（快照），保证历史审查可复现。
2. **全程溯源**：`parse_segment` 作为溯源锚，审查结论经 `finding_evidence` 回溯到原文页码。
3. **防篡改**：`review_round.snapshot_hash` 存材料+抽取的有序 SHA256。
4. **语义检索**：条款、规则版本、写作范式、问题模式均带 embedding 字段，支持向量检索。
5. **文件统一桥**：所有文件经 `file_object` 登记，业务表仅存 `file_id`。
6. **日志分区**：三张日志表按月 RANGE 分区、放弃外键，适配 append-only 高写入。
7. **安全合规**：密级（secrecy_level）、保密资质（security_qual_level）、失信库（integrity_record）贯穿用户、单位、材料。
