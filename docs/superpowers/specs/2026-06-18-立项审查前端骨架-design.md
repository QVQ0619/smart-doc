## 2. 本期范围

- 三栏布局骨架（左菜单 / 中工作区 / 右对话），浅色专业风（对齐 demo）。

## 4. 布局

```
┌─────────┬──────────────────────────┬──────────────┐
│ 菜单栏   │        中间工作区          │  Agent 对话   │
│ (固定)   │   (路由切换业务占位页)      │  (ChatView)   │
│ ~220px  │        flex:1            │  ~380px(可折叠) │
└─────────┴──────────────────────────┴──────────────┘
```

- 外层 `AppShell`：CSS Grid 三列，整页 `height:100vh`，各列独立滚动。
- 右栏满足 ChatView 布局要求：外层容器 `height` + `min-height:0` + `display:flex` + `flex-direction:column` + `overflow:hidden`。
- 右栏支持折叠/展开：折叠时用 CSS 隐藏 header+body、显示窄竖条，**不卸载** SessionGate / ChatView（保留会话与 socket 订阅，参考 blade-coa）。折叠态用 localStorage 持久化。

## 6. 导航与路由

### 菜单结构（最终版）

| 分组 | 菜单项 | route key | 占位页用途（后续填充） |
| --- | --- | --- | --- |
| 概览 | 工作台 | `home` | 概览统计、最近审查、快捷入口 |
| 审查 | 新建审查 | `review-new` | 上传立项材料、选规则/配置、发起审查 |
| 审查 | 审查任务 | `review-tasks` | 进行中 / 历史批次看板 |
| 审查 | 审查报告 | `review-report` | 审查结果、问题清单、专家意见 |
| 资源库 | 规则库 | `lib-rules` | 审查规则的增删改查 |
| 资源库 | 配置包 | `lib-packs` | 规则 + 专家配置的打包管理 |
| 资源库 | 审查文档库 | `lib-archive` | 已归档的审查材料与报告 |
| 系统 | 关于流程 | `about` | 审查流程说明（静态页） |

### 路由机制
- `useRouteStore`（zustand）保存 `route` 与 `setRoute`，默认 `home`。
- `App.tsx` 中按 `route` 渲染对应页面组件（switch/map 表）。
- 不引入 react-router（与 blade-coa 一致，轻量；如未来需要深链再加）。

### 占位页
- 每个页面 = `PageScaffold`（页头：标题 + 一句说明）+ `EmptyState`（"功能建设中"图标 + 文案）。
- 视觉统一，保证骨架完整；后续逐页把 `EmptyState` 替换为真实内容。

## 7. Blade 对话集成

### 环境配置（`.env`）
```
VITE_BLADE_API_BASE=http://115.190.152.1:8020
VITE_BLADE_TOKEN=
# 可选，后续填：
# VITE_BLADE_SOLUTION_ID=
# VITE_BLADE_BIZ_ROLE_ID=
```
- `VITE_BLADE_TOKEN` 留空，由使用者填 `sk-blade-v2-...`。
- `blade/config.ts` 读取 env，`VITE_BLADE_API_BASE` 缺省回退到 `http://115.190.152.1:8020`；`baseUrl` 必须是 origin，不带 pathname。

### Provider 装配（`main.tsx`）
- `QueryClientProvider`（react-query）
- `BladeClientProvider`：`baseUrl`、`token={() => getToken()}`（getter 形式，便于运行时切换）
- `ConfigProvider`（antd 主题）+ sonner `<Toaster>`

### ChatPanel 行为
- 挂载时若无活跃会话，自动 `createSession(intent)` 建一个会话，并 `setActiveSession`。
- 渲染 `<ChatView sessionId={sessionId} />`，发送默认 `mode: "executing"`（业务直接干活）。
- 顶部"＋ 新建会话"按钮：重新 `createSession` 切换。
- 折叠/展开按钮：见 §4。
- 创建会话暂不指定 `solution_id` / `biz_role_id`（用系统默认）；env 变量已预留，填了就透传。

### 预留扩展点（本期仅注释/接口占位，不接业务）
- 宿主 → Agent：`useUiBridgeStore`（注入上下文 pill、追加输入、触发发送）。
- Agent → 宿主：`_meta.bridge` action 驱动中间页面（如刷新某页数据）；`_meta.ui` 渲染可视化卡片（ChatView 自动处理 iframe 卡片）。
- 文件上传：`client.sessions.uploadFiles(sessionId, ".", files)` —— 后续"新建审查"页接入。

- 浅色专业风：白 / 浅灰背景，蓝色主色，卡片化、留白克制，对齐 demo 与政务/审查严肃场景。
- 通过 antd `ConfigProvider` 的 theme token 统一（主色、圆角、字号），全局样式在 `styles/global.css`。
- 三栏分隔线、菜单分组标题、页头样式保持一致。

## 9. 验收标准

1. `pnpm dev`（或 npm）启动后，浏览器呈现三栏布局，无报错。
2. 左菜单 8 个菜单项可点击切换，中间区渲染对应占位页（页头 + 空状态），视觉统一。
3. 在 `.env` 填入有效 `VITE_BLADE_TOKEN` 后，右侧对话面板能自动建会话、发送消息并收到 Agent 流式回复；可"新建会话"。
4. 右栏可折叠/展开，折叠后再展开会话与历史不丢失。
5. token 留空时，对话面板有清晰的"未配置 API Key"提示，不白屏崩溃。
6. 目录结构与 §5 一致，类型检查（`tsc -b`）与构建（`vite build`）通过。

## 10. 风险与注意

- agent-kit 子入口导入规则、React 19 强约束（见 §3 注释）。
- ChatView 外层容器布局要求（见 §4），否则消息列表不滚动。
- Zustand selector 不要每次返回新数组/对象（React 19 下会无限渲染），用稳定空常量。
- baseUrl 不能带 pathname（socket.io namespace 问题）。
- 本项目当前非 git 仓库；如需版本管理可后续 `git init`。
