# 本地 skill 按会话推送入库 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `save-rule-doc` 技能源放仓库；前端建会话成功后自动 `uploadSessionSkill` 把它推到该会话，agent 自动发现并在用户传文件+说「这是规则文件」时调用它经隧道入本地规则库。

**Architecture:** 前端用 SDK 的 `partnerSkillApi.uploadSessionSkill(sessionId, {name, files})` 把 SKILL.md + 脚本 + api_base(由 `VITE_SMART_DOC_API` 注入) 推到会话沙箱。脚本(`backend/agent_shim/smart_doc_add.py`)与后端接口零改动。SKILL.md 与脚本由 Vite `?raw` 作为字符串打进前端构建。

**Tech Stack:** React 19 + TS + Ant Design 5 + sonner + `@blade-hq/agent-kit@0.5.11`(`/react` 导出 `partnerSkillApi.uploadSessionSkill`) + vitest + Vite `?raw` 导入。

## Global Constraints

- 技能名必须匹配 `^[a-z0-9-]+/[a-z0-9-]+$`，固定用 `"local/save-rule-doc"`（连字符，不能下划线）。
- `uploadSessionSkill(sessionId, payload)` 的 `payload = { name: \`${string}/${string}\`, files: { path: string; content: string }[] }`；返回 `{ name, skill_dir, file_count, overwritten }`。
- 后端地址来自 `import.meta.env.VITE_SMART_DOC_API`（trim，空→视为未配置）；作为 `scripts/api_base.txt` 内容推送。换隧道域名只改 `.env` 重启，不动 skill。
- **解耦**：`pushRuleDocSkill` 内部全程 try/catch，任何失败都不得抛出阻断会话/聊天；失败 `toast.warning`。
- 通知用 `sonner` 的 `toast`。
- 前端命令 CWD = 仓库根；测试套件较慢（全量约 1~1.5 分钟），命令给足超时（全量 ≥240s，单文件 ≥120s）。
- 现有前端 27 测试不回归；后端零改动（不碰 `backend/`）。
- `?raw` 导入路径：从 `src/blade/` 到 `../../backend/agent_shim/smart_doc_add.py?raw` 与 `../../blade/skills/save-rule-doc/SKILL.md?raw`，均在仓库根内（Vite/vitest 默认 `server.fs.allow` 覆盖）。`?raw` 模块类型由 `vite/client` 提供（`src/vite-env.d.ts` 已 `/// <reference types="vite/client" />`）。
- 不要 `git add -A`：仓库有未跟踪的 `docs/file/`、`docs/参考文件/`、`exports/`、`.tools/`，逐文件/逐目录 `git add`。

# save-rule-doc — 保存规则文件到规则库

本技能随会话下发一个纯标准库脚本 `scripts/smart_doc_add.py`，它把 sandbox 里的文件
multipart 上传到立项审查后端的 `POST /api/standard-docs` 完成入库。后端地址在同目录的
`scripts/api_base.txt`（由前端注入）。

## 何时触发
用户在聊天中**已用“+”上传了一个文件**（文件已在当前 session 的 sandbox 中），并表达把它
存为规则文件的意图，例如：“这是规则文件”“把这个存为规则文件”“加入规则库 / 保存到规则库”。

## 前置判断
1. 确认 sandbox 中确有用户刚上传的文件，拿到它的**绝对路径**。
2. 判断它是否为**规则类文档**（政策、指南、规范、办法等）。明显不是则先说明并询问，不擅自入库；意图不明先澄清。

## 执行
对确认要保存的文件，用 Bash 工具运行：

```bash
# 1) 定位随本技能下发的脚本
SCRIPT="$(find / -path '*save-rule-doc/scripts/smart_doc_add.py' 2>/dev/null | head -1)"
# 2) 读取后端地址（前端注入到脚本同目录的 api_base.txt）；tr 去掉可能的 CR/LF
API="$(cat "$(dirname "$SCRIPT")/api_base.txt" | tr -d '\r\n')"
# 3) 上传入库（每个文件绝对路径作为一个参数，可多个）
SMART_DOC_API="$API" python3 "$SCRIPT" "<sandbox 中的文件绝对路径>"
```

> 若 `find` 没找到脚本，如实告知“技能脚本缺失，无法入库”，不要伪造结果。

## 解读结果（按退出码）
| 退出码 | 含义 | 你应如何回报 |
|---|---|---|
| 0 | 全部成功 | 报告每个文件的标题与编号，例：“已存入规则库：政策A（SD-abc123）” |
| 1 | 用法错误（没给文件参数） | 说明没有可保存的文件 |
| 2 | 文件不存在/不可读 | 说明没找到文件，请用户确认是否已上传 |
| 3 | 连不上规则库后端 | 说明后端暂时不可达（可能隧道未开/地址错），建议稍后重试 |
| 4 | 后端返回错误（非 2xx） | 说明入库失败（附 stderr 原因），不要声称成功 |
| 5 | 部分/全部文件失败 | 逐个报告：成功的报标题+编号，失败的报 stderr 里的原因（如超过大小上限） |
| 6 | 后端地址未配置 | 部署问题：`api_base.txt` 为空，提示需在前端配 `VITE_SMART_DOC_API` |

## 铁律
- **只有退出码 0 且 stdout 出现 `doc_code=...` 才能说“已保存成功”。** 其他情况一律如实说明失败原因，绝不谎报。
- 不要凭空编造 `doc_code` 或标题——只转述脚本的实际输出。
- 不得绕过本脚本直连数据库或自行拼 SQL；所有入库一律走该脚本（它内部调后端的带校验接口）。

- [ ] **Step 4: 创建 `src/blade/sessionSkill.ts`**

```ts
import { toast } from "sonner";
import { partnerSkillApi } from "@blade-hq/agent-kit/react";
import SKILL_MD from "../../blade/skills/save-rule-doc/SKILL.md?raw";
import SHIM_PY from "../../backend/agent_shim/smart_doc_add.py?raw";

const SKILL_NAME = "local/save-rule-doc";

export function getSmartDocApi(): string | undefined {
  const v = import.meta.env.VITE_SMART_DOC_API?.trim();
  return v && v.length > 0 ? v : undefined;
}

/**
 * 把 save-rule-doc 技能推送到指定会话（agent 沙箱）。best-effort：
 * 任何失败只 toast.warning，绝不抛出阻断会话/聊天。
 */
export async function pushRuleDocSkill(sessionId: string): Promise<void> {
  try {
    const apiBase = getSmartDocApi();
    if (!apiBase) {
      toast.warning("未配置 VITE_SMART_DOC_API，agent 入库会失败（请在 .env 配置隧道域名后重启）");
    }
    const res = await partnerSkillApi.uploadSessionSkill(sessionId, {
      name: SKILL_NAME,
      files: [
        { path: "SKILL.md", content: SKILL_MD },
        { path: "scripts/smart_doc_add.py", content: SHIM_PY },
        { path: "scripts/api_base.txt", content: apiBase ?? "" },
      ],
    });
    // eslint-disable-next-line no-console
    console.info(
      "[sessionSkill] pushed save-rule-doc ->",
      res.skill_dir,
      `(files=${res.file_count})`,
    );
  } catch (e) {
    toast.warning("规则文件技能推送失败：" + (e instanceof Error ? e.message : String(e)));
  }
}
```
